# Защита Meridian — шпаргалка (7 мин + 5 Q&A)

## Питч (30 сек)
Meridian теряет выручку (−11.5%/−5.3% г/г) и клиентов (отток к AI-инструментам). Аналитика — 5–10 дней через тикеты. Мы построили **мультиагентного AI-аналитика для совета директоров**: вопрос на русском → обоснованный ответ с цифрами, методом, допущениями и графиком за секунды. Главное отличие — **инженерная честность**: система говорит «данных недостаточно», а не выдумывает.

## Архитектура (1 мин)
```
Вопрос → Planner (режим: BI / research / insufficient)
       → Extractor (метрика из библиотеки ИЛИ guarded SQL → DuckDB)
       → Analyst (цифры → бизнес-вывод на русском)
       → Critic (чек-лист ловушек, loopback ≤2)
       → Visualization (тип графика)
       → ответ {response, assumptions, trace, chart, insufficient_data}
```
- **Node.js/TS**, Fastify, **OpenAI-совместимый API → YandexGPT 5.1**, DuckDB (read-only, guard-rails).
- **3 режима кейса:** Диалоговый BI (контекст сессии) · Ad-hoc исследование (декомпозиция) · Отчёты по расписанию (cron → дашборд здоровья → Telegram).

## Цифры (увязка с критериями оценки)

| Критерий (вес) | Чем закрыт | Доказательство |
|---|---|---|
| **Точность 30%** | библиотека из 13 проверенных метрик + guarded SQL | LLM-судья: 11 correct / 2 partial / **0 wrong**, средний **98/100** |
| **Boundary discipline 15%** | planner ловит невозможное, Critic честность | банк вопросов: **18/18** верный флаг (5/5 ловушек отклонены) |
| **Интерпретация 25%** | Analyst: «цифры + метод + допущения» | напр. SMB LTV/CAC 0.59 → «привлечение убыточно»; + блок рекомендаций |
| **Архитектура 20%** | 4 агента + planner, контракты, Critic-loopback | + робастность: стресс-тест **0/13 провалов** |
| **Визуализация 10%** | Chart.js в вебе + Unicode-бары в Telegram | дашборд + отчёт с графиками |

## Стратегические находки (этап 1) — чем удивить
- 🔻 **Не падение спроса, а монетизации:** выручка ↓, но **GMV растёт** (120→139→165 млрд); take rate проседает (~5.5%), EBITDA в 2025 отрицательная.
- 💸 **SMB убыточен в привлечении:** LTV/CAC — Large 3.03 / Mid 1.68 / **SMB 0.59**.
- 🤖 **AI-угроза реальна:** 14.7% оттока — `ai_alternative`; тег `ai_competitor` в NPS.
- 🔴 **Ловушка данных:** `orders` ≠ `financials` (расходятся ~4.7×) — система это знает и не смешивает (правило в Critic).

## Демо-сценарий (3–4 мин)
1. **Диалоговый BI:** «Какой NPS по продуктовым линиям?» → ответ + bar-chart + «Допущения»/«Трасса агентов» (показать инженерную прозрачность). Затем уточнение «а где самый низкий?» — **контекст держится**.
2. **Boundary discipline:** «Какой прогноз выручки на 2027?» → честное «недостаточно данных» (не выдумка).
3. **Ad-hoc исследование:** сложный вопрос → показать **декомпозицию** на под-вопросы + синтез.
4. **Отчёты:** «Собрать сейчас» → дашборд здоровья + **доставка в Telegram** (графики + рекомендации + легенда). Упомянуть cron «пн 9:00».
5. **Робастность (если спросят):** prompt injection → система отказывается, не раскрывает промпт.

## Осознанные ограничения (честно — та же дисциплина к себе)
- Сессии и инбокс отчётов — **in-memory** (перезапуск очищает); продакшен → Redis/БД.
- Графики в Telegram — **Unicode-бары** (нативный PNG-рендер не ставится в окружении; внешние сервисы запрещены). В вебе — полноценный Chart.js.
- Один тир LLM (YandexGPT 5.1) на всех агентов; можно удешевить planner/viz на Lite.
- Нет аутентификации/мультитенантности (вне scope хакатона).
- `orders`/`financials` — независимые синтетические слои; кросс-сверку не делаем осознанно.

## Q&A — заготовки
- **«Почему не выдумывает?»** — planner классифицирует вне-скоуп → insufficient; Critic сверяет числа ответа со строками SQL; guard-rails read-only + только SELECT.
- **«Как считаете точность?»** — LLM-судья сверяет ответ с эталонным SQL из каталога метрик (этап 4), 0 wrong на 18 вопросах.
- **«Масштабирование?»** — stateless-агенты за Fastify; DuckDB→Postgres/Yandex Cloud; сессии→Redis.
- **«Почему YandexGPT?»** — OpenAI-совместимый клиент, переключается одной env-строкой; выбран Yandex под кейс/деплой.
- **«Что если данных частично нет?»** — research-режим: отвечает по тому, что есть, помечает пробел; флаг insufficient только если данных нет совсем.

## Деплой
Развёртывание в Yandex Cloud: контейнер с `npm start`, DuckDB-файл или миграция на Managed PostgreSQL; env: LLM_*, SCHEDULE_CRON, TELEGRAM_*. (Не выполнено — следующий шаг.)
