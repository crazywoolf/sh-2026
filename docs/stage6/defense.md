## Питч (30 сек)
Meridian теряет выручку и клиентов (уходят к AI-инструментам), а аналитика отвечает совету **5–10 дней через тикеты**. Мы построили **мультиагентного AI-аналитика для совета директоров**: вопрос на русском → обоснованный ответ с **цифрами, методом, допущениями и графиком** за секунды. Главное отличие — **инженерная честность**: система говорит «данных недостаточно» и **опровергает удобные управленческие тезисы цифрами**, а не поддакивает.

## Архитектура (1 мин)
```
Вопрос → Planner (режим: BI / research / insufficient)
       → Extractor (метрика из библиотеки ИЛИ guarded SQL → DuckDB read-only)
       → Analyst (цифры → бизнес-вывод на русском, скептический лид)
       → Critic (чек-лист ловушек, loopback ≤2)  ← ключевой для оценки
       → Visualization (тип графика)
       → {response, assumptions, trace, chart, insufficient_data, session_id}
```
- **Node.js/TS · Fastify · OpenAI-совместимый API → YandexGPT 5.1 · DuckDB read-only + guard-rails** (только SELECT, белый список таблиц).
- **Оркестратор:** держит контекст диалога; в research-режиме гоняет под-вопросы **параллельно** и синтезирует; гарантия — **никогда не 500**.
- **3 режима:** Диалоговый BI (контекст) · Ad-hoc исследование (декомпозиция) · Отчёты по расписанию (cron → дашборд здоровья → инбокс/Telegram).

## Главный дифференциатор — скептическая дисциплина (на этом делаем акцент)
Тестер жюри бьёт **непрямыми скептическими** вопросами («X выросло — значит ли это…?», «это действительно так?»). Система их **узнаёт по смыслу** (не по ключевым словам) и **опровергает цифрами**:
- «рост оборота → больше зарабатываем?» → **Нет:** GMV +37%, но выручка −16%, take rate 6.89%→4.24% (проблема монетизации).
- «NPS вырос → продукт лучше?» → **Не обязательно:** смещение выжившей когорты — детракторы ушли с **27.7%→13%**, а не продукт улучшился.
- «отток упал вдвое → база стабильнее?» → **Формально да (3909→1863), но нет:** доля спящих (экономический отток) растёт **7.2%→12%**.

Это работает и на **незнакомых** аспектах — в Analyst встроен список известных напряжений Meridian (NPS≠отток, усреднение прячет SMB, доля≠значимость, сегмент vs GMV, точка vs тренд, маржа≠прибыль) → дисциплина переносится, а не зазубрена.

## Стратегические находки — чем удивить
- 🔻 **Не падение спроса, а монетизации:** выручка ↓, но **GMV растёт** (120→139→165 млрд); take rate 6.89%→4.24%; EBITDA в 2025 отрицательная.
- 💸 **SMB убыточен в привлечении:** LTV/CAC — Large 3.03 / Mid 1.68 / **SMB 0.59**.
- 📊 **«Уйти в Large» режет оборот:** у Large лучшая экономика, но лишь **25% GMV** (Mid 39.6% + SMB 35.2% = ¾).
- 🤖 **Данные опровергают интуицию по оттоку:** №1 причина — **цена 24.6%**, AI лишь **14.7%** (не «все ушли к AI»).
- 🔴 **Ловушка данных:** `orders` ≠ `financials` (расходятся ~4.7×) — Critic это знает и не смешивает.

- **Мониторинг `/monitor`** на первом прогоне жюри **выловил точные вопросы тестера** → мы за минуты диагностировали пробелы → закрыли → редеплой.

Проверяет **числа и поведение** (не оценку LLM): числа из SQL, маркеры, флаг insufficient. Гоняется против прода → красное/зелёное после любой правки. **40/40 на проде.**

## Демо-сценарий (3–4 мин) — вести через `/demo`
1. **`/demo`** — слайды «от клиента»: проблема совета → **4 кликабельные гипотезы** (клик открывает живой чат с вопросом!) → карта компромиссов.
2. **Скептический вопрос вживую:** «Рост оборота — значит, бизнес больше зарабатывает?» → **«Нет»** + годовые цифры (это вау-момент).
3. **Boundary discipline:** «Разбей выручку по полу клиентов» → честный отказ + причина (B2B/юрлица); «Какой месяц лучший?» → уточнение критерия + варианты.
4. **Ad-hoc:** «Здоров ли бизнес?» → декомпозиция на 4 оси + синтез.
5. **Прозрачность:** раскрыть «Трасса агентов» / «Допущения» под любым ответом.

## Осознанные ограничения (честно — та же дисциплина к себе)
- Сессии и инбокс отчётов — **in-memory** (перезапуск очищает); мониторинг дублируется в **JSONL на диске**. Прод → Redis/БД.
- Графики в Telegram — Unicode-бары (нативный PNG не ставится в окружении). В вебе — полноценный Chart.js.
- Один тир LLM (YandexGPT 5.1) на всех агентов; planner/viz можно удешевить на Lite.
- `orders`/`financials` — независимые синтетические слои; кросс-сверку не делаем осознанно.
- Связь NPS↔отток по линиям — показываем разброс NPS + оговорку (прямой разбивки оттока по линиям в данных нет).

## Q&A — заготовки
- **«Почему не выдумывает?»** — Planner классифицирует вне-скоуп → insufficient; Critic сверяет числа ответа со строками SQL и ловит раздутые join'ы; guard-rails read-only + только SELECT.
- **«Не подгонка ли под утёкшие вопросы?»** — нет: усилили **общий** механизм (список напряжений), проверили на **6 новых** скептических вопросах про другие аспекты — **6/6** в регресс-сьюте; маршрутизация семантическая, не по словам.
- **«Как меряете точность?»** — груз-true регресс-сьют: числа прямо из SQL + флаг поведения, 40/40; плюс пул из 32 вопросов совета (23 good/5 partial/1 poor).
- **«Масштабирование?»** — stateless-агенты за Fastify; DuckDB→Postgres; сессии→Redis; параллельный research уже есть.
- **«Почему YandexGPT?»** — OpenAI-совместимый клиент, переключается одной env-строкой; выбран Yandex под кейс/деплой.
