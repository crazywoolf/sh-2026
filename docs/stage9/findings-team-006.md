## №1 — NPS по сегментам
**Вопрос:** «Какой NPS по каждому сегменту (Large, Mid, SMB)?»
**Ответ:** «В предоставленных данных нет информации об NPS по сегментам. Есть только выручка по сегментам…»
**Что не так:** Ложный отказ — NPS по сегментам считается (опросы + сегмент клиента).
**Как должно быть:** Large **45.3**, Mid **23.5**, SMB **22.1**.
**SQL:** `SELECT c.segment, round(100.0*(count(*) FILTER(WHERE n.score>=9)-count(*) FILTER(WHERE n.score<=6))/count(*),1) FROM nps_responses n JOIN customers c USING(customer_id) GROUP BY 1;`

## №5 — доля GMV по сегментам
**Вопрос:** «Какая доля GMV приходится на каждый сегмент?»

**Как должно быть:** Mid 39.6%, SMB 35.2%, Large 25.3%. `SELECT c.segment, round(100.0*sum(o.gmv)/sum(sum(o.gmv)) OVER(),1) FROM orders o JOIN customers c USING(customer_id) GROUP BY 1;`

## №11 — LTV/CAC у SMB
**Вопрос:** «Какой LTV/CAC у сегмента SMB?»
**Ответ:** «Невозможно рассчитать LTV/CAC для SMB, факты содержат только количество ячеек (59)…»
**Что не так:** Ложный отказ — LTV/CAC SMB считается напрямую.
**Как должно быть:** SMB LTV/CAC = **0.59** (последний месяц; убыточен). `SELECT round(avg(ltv_12m)/avg(cac),2) FROM unit_economics_monthly WHERE segment='SMB' AND month=(SELECT max(month) FROM unit_economics_monthly);`

## №12 — месяц с максимальной выручкой
**Вопрос:** «В каком месяце была максимальная выручка?»
**Ответ:** «Невозможно определить — факты содержат показатели только за декабрь 2025…»

## №13 — NPS по годам
**Вопрос:** «Какой NPS по годам?»
**Ответ:** «Общий NPS вырос с **19.1** (2023) до 28.7 (2024) и **35.6** (2025)…» (когорта 54.2→34.9)
**Что не так:** Завышенные значения. Стандартный NPS (%промо−%детр) = **17.1 / 26.4 / 34.0**; когорта 52.5→33.4. (Survivorship-вывод верный, но числа неточны.)
**Как должно быть:** 17.1 / 26.4 / 34.0. `SELECT year(response_date) yr, round(100.0*(count(*) FILTER(WHERE score>=9)-count(*) FILTER(WHERE score<=6))/count(*),1) FROM nps_responses GROUP BY 1 ORDER BY 1;`

## №14 — выручка по продуктовым линиям
**Вопрос:** «Какая выручка по продуктовым линиям?»
**Ответ:** «Разработка и IT — 270,0 млн, Маркетинг — 198,6 млн, Юридические — 114,3 млн. Это топ-3; **данные по остальным линиям отсутствуют**.»
**Что не так:** Неверно «остальные отсутствуют» — в витрине **все 9** продуктовых линий с выручкой/заказами; система обрезала вывод до 3 и ложно заявила отсутствие остальных.
**Как должно быть:** 9 линий (Разработка и IT, Маркетинг, Юр.услуги, Консалтинг, Рекрутинг, Дизайн, Логистика, Бухгалтерия, Аутсорс). `SELECT count(*) FROM product_lines;` → 9.

## №15 — динамика спящих по годам
**Вопрос:** «Как менялась доля спящих (неактивных) клиентов по годам?»
**Ответ:** «Общий NPS вырос: 19.1 (2023) → 35.6 (2025)…» (целиком про NPS)
**Что не так:** Ответ не на вопрос — вместо динамики спящих выдан рассказ про NPS (вывод без опоры на заданный вопрос).
**Как должно быть:** Доля неактивных (dormant+churning) на конец года: 18.4% → 22.8% → 32.5%. `SELECT year(month) yr, round(100.0*count(*) FILTER(WHERE status IN('dormant','churning'))/count(*),1) FROM customer_activity_monthly WHERE month(month)=12 GROUP BY 1;`
