# Транскрипт работы агентов Meridian

**Тест-кейс (вопрос):** Сравни динамику GMV и чистой выручки по годам. Что с монетизацией и take rate?

Документ показывает, **как система пришла к выводу** — по шагам конвейера агентов. Каждый этап снабжён пояснением «что делает агент» (роль) и «что произошло здесь» (интерпретация конкретного шага).

```
Вопрос → Planner → Extractor → Analyst → Critic → Visualization → Ответ
```
> Сквозная гарантия: система **никогда не возвращает 500** и **не выдумывает данные** — если их нет, честно помечает `insufficient_data`.

---

## 1. Planner — классификация и декомпозиция
> **Роль агента:** понимает вопрос и решает РЕЖИМ обработки: `bi` (один срез данных), `research` (декомпозиция на под-вопросы) или `insufficient` (нужного поля нет в витрине). Для сложных вопросов разбивает их на самодостаточные под-вопросы.

```json
{
  "mode": "bi",
  "reasoning": "Вопрос требует сравнения двух метрик (GMV и чистая выручка) и анализа связанных показателей (take rate и монетизация) из одной таблицы financials_monthly. Это единый срез данных с разбивкой по годам, не требуется собирать информацию из разных источников.",
  "sub_questions": [
    "Сравни динамику GMV и чистой выручки по годам. Что с монетизацией и take rate?"
  ]
}
```

**→ Что произошло:** выбран режим **`bi`** (диалоговый BI — отвечается одним срезом данных); вопрос разложен на **1** под-вопрос(а). Дальше каждый под-вопрос проходит Extractor → Analyst → Critic.

## 2. Extractor — извлечение данных (вопрос → SQL → DuckDB)
> **Роль агента:** превращает под-вопрос в данные. Берёт готовую **проверенную метрику** из библиотеки, либо пишет безопасный `SELECT` (read-only, guard-rails: только чтение, белый список таблиц). Сам определяет, достаточно ли данных.

- **под-вопрос:** Сравни динамику GMV и чистой выручки по годам. Что с монетизацией и take rate?
- **подход:** готовая метрика `monetization_by_year`
- **данных достаточно:** да · **получено строк:** 3
- **SQL (исполнен на DuckDB read-only):**
```sql
SELECT year(month) AS year,
                 round(sum(gmv)) AS gmv,
                 round(sum(revenue_net)) AS revenue_net,
                 round(100.0*avg(take_rate), 2) AS take_rate_pct
          FROM financials_monthly GROUP BY 1 ORDER BY 1
```
- **первые строки результата:**

**→ Что произошло:** данные получены (3 строк) — Extractor передаёт их Analyst для интерпретации.
